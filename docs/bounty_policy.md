# Uptime Bounty Policy (Phase 3)

## Objective
Reward operators that maintain stable, high‑availability nodes.

## Eligibility
- Node must be registered in governance state.
- Node must publish anchors and participate in quorum.

## Measurement
- Uptime is measured via Prometheus metrics and health checks.
- Minimum target: 90% availability over a rolling 30‑day window.
- Disqualifiers:
  - Extended downtime without notice
  - Repeated invalid envelopes
  - Slashed stake

## Reporting
Operators should provide:
- Node ID
- Public key
- Metrics endpoint
- Region

## Payout
Payouts are scheduled monthly and derived from fee pool totals.
